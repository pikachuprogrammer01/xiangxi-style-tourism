# 湘西风情 — 产品需求文档（PRD）

## 1. 项目概况

| 属性 | 说明 |
| :--- | :--- |
| 项目名称 | 湘西风情 —— 地方文化与旅游宣传平台 |
| 技术栈 | Vue 3（Composition API）、Element Plus、Vite、JavaScript、CSS3 |
| 设计目标 | 构建响应式、视觉美观、内容充实的湘西旅游文化宣传站点 |
| 项目性质 | 专科毕业设计（前端方向） |
| 核心定位 | 以数字化窗口展示湘西自然风光、非遗文化、民俗美食，提供沉浸式浏览体验 |

## 2. 用户分析

### 2.1 目标用户画像

| 用户类型 | 特征 | 核心需求 | 使用场景 |
| :--- | :--- | :--- | :--- |
| 外地游客 | 计划前往湘西旅游，20-45岁，手机/PC混合查询 | 景点推荐、行程规划、美食攻略 | 出行前做攻略、旅途中查信息 |
| 本地居民 | 湘西本地或周边居民 | 发现新景点、分享家乡文化 | 周末休闲、向外地朋友推荐 |
| 文化爱好者 | 对少数民族文化、民俗风情感兴趣的人群 | 深入了解非遗文化、传统手工艺、节庆活动 | 文化探索、学术研究 |

### 2.2 用户需求优先级

- **P0 核心需求**：浏览景点信息、查看美食推荐、了解非遗文化、查看旅游攻略
- **P1 重要需求**：用户注册登录、收藏内容、浏览历史、响应式适配移动端
- **P2 增强需求**：留言板互动、摄影画廊浏览、搜索筛选

### 3.2 各页面详细功能

#### 3.2.1 首页（Home）

**定位**：网站门面，承担流量分发和第一印象塑造。

**功能模块**：

- **顶部导航栏**：LOGO（点击回首页）、主导航菜单、用户头像/登录按钮，滚动时固定顶部 + backdrop-filter 模糊
- **全屏轮播图**：100vw × 70vh（桌面）/ 50vh（移动），3-5 张湘西风景图，每张配标题+副标题叠加，自动轮播 5s，支持指示器/箭头切换，悬停暂停
- **热门推荐区**：3-4 个景点卡片（图片+名称+简介+评分），悬停图片放大 + 阴影加深
- **文化特色区**：3 列布局（民俗文化、传统手工艺、节庆活动），图标+标题+描述
- **美食速览区**：横向滚动 4-6 个美食缩略图，点击跳转美食详情
- **页脚**：版权信息 + 简要导航链接。`© 2026 湘西风情 版权所有`

**非功能需求**：首屏加载 < 3s，图片懒加载，键盘 Tab 可导航。

#### 3.2.2 登录页（Login）

- 卡片居中，最大宽度 400px，渐变背景
- 表单：用户名/邮箱 + 密码（可切换显示/隐藏）+ 记住我
- 验证：用户名 3-20 字符，密码 6-20 字符含字母和数字，blur 实时校验
- 登录成功：Message 提示，1.5s 后跳转；登录失败：提示错误，密码清空
- 底部链接："还没有账号？立即注册"
- 已登录用户访问自动重定向到首页

#### 3.2.3 注册页（Register）

- 卡片居中，最大宽度 450px
- 表单：用户名 + 邮箱 + 密码 + 确认密码 + 手机号（选填）
- 密码强度指示器（弱/中/强三档，颜色红/黄/绿）
- 用户名唯一性检测、邮箱格式校验、两次密码一致性校验
- 注册成功跳转登录页；底部链接："已有账号？立即登录"

#### 3.2.4 景点总览页（Attractions）

- **左侧边栏 200px**：分类筛选（全部/自然风光/历史古镇/民族村寨/红色旅游）、排序选项、价格滑块
- **右侧内容区**：搜索栏 + 结果统计，CSS Grid 卡片列表（桌面 3 列 / 平板 2 列 / 手机 1 列），分页器（每页 9 个）
- **卡片内容**：16:9 图片 + 名称 + 评分（el-rate）+ 标签（el-tag）+ 简介截断 + 价格 + 查看详情/收藏按钮
- **空状态**：无搜索结果时显示插画 + 提示文字

#### 3.2.5 景点详情页（AttractionDetail）

动态路由 `/attraction/:id`，根据 ID 渲染不同景点内容。

- **左侧边栏 200px**：开放时间、游玩时长、最佳季节、门票价格、交通指南；收藏/分享按钮；3 个相关推荐缩略图
- **右侧主内容区**：顶部 400px 主图 + 标题叠加；面包屑导航；名称 + 评分 + 地址 + 标签；Tab 切换（景点介绍 / 游玩攻略 / 用户评价 / 交通指南）

#### 3.2.6 美食推荐页（Foods）

- 页面标题区："湘西味道" + 副标题
- 分类筛选栏：全部 / 特色主食 / 风味小吃 / 传统饮品 / 节日美食（横向滚动标签）
- 响应式网格：桌面 4 列 / 平板 3 列 / 手机 2 列，gap 24px
- 美食卡片：正方形图片 + 名称 + 一句话简介 + 口味标签 + 热度指示
- 加载更多：初始 8 个，点击追加 4 个，底部 "已经到底啦~"

#### 3.2.7 美食详情页（FoodDetail）

动态路由 `/food/:id`，根据 ID 渲染不同美食内容。

- 顶部全宽美食主图（350px）+ 名称叠加
- 面包屑导航：首页 > 美食推荐 > 当前美食
- 两栏布局：左侧 70%（美食故事、食材介绍、制作步骤、口味特点、推荐店铺）、右侧 30%（大图卡片、营养信息、相关推荐、"我吃过"/收藏按钮）

#### 3.2.8 旅游攻略页（TravelGuide）

- 最佳旅游季节：4 个季节卡片，每个含季节名称 + 特点描述
- 推荐路线：多条路线卡片（标题 + 天数 + 简介 + 详细日程列表）
- 旅行贴士：交通/住宿/饮食/天气分类，每个含多条实用建议

#### 3.2.9 非遗展示页（Heritage）

- 分类筛选按钮组（全部 / 传统技艺 / 民俗活动 / 传统美术 / 传统音乐等）
- 响应式网格卡片：非遗项目图片 + 名称 + 级别标签（国家级/省级，不同颜色）+ 列入年份
- 卡片整体可点击，跳转非遗详情页

#### 3.2.10 非遗详情页（HeritageDetail）

动态路由 `/heritage/:id`。

- 顶部全宽封面图 + 暗色叠加层 + 名称 + 级别标签 + 列入年份
- 面包屑导航：首页 > 非遗展示 > 当前项目
- 详细介绍区：项目概述 + 历史渊源 + 技艺特点 + 传承现状 + 文化价值

#### 3.2.11 摄影画廊页（Gallery）

- 分类筛选按钮（全部 / 自然风光 / 人文纪实 / 苗寨风情等）
- 响应式图片网格（Masonry 风格，不同高度交错排列）
- 点击图片弹出灯箱（lightbox）：全屏遮罩 + 大图 + 标题 + 作者 + 左右切换箭头 + 关闭按钮
- 键盘支持：Esc 关闭、左右箭头切换

#### 3.2.12 留言板页（Guestbook）

- 留言发布表单：昵称输入 + 留言内容 textarea + 发布按钮
- 前端校验：昵称和内容不能为空
- 留言列表：头像（DiceBear API 生成）+ 昵称 + 时间 + 内容 + 点赞按钮
- 新留言插入列表顶部

#### 3.2.13 个人中心（Profile）

**需登录访问**，未登录重定向到 `/login?redirect=/profile`。

- **顶部统计卡片行（4 列）**：收藏数 / 浏览历史数 / 评论数 / 账号等级
- **左侧个人信息卡片（280px）**：头像 + 用户名 + 注册时间 + 个人简介（可编辑）+ 账号绑定状态 + 编辑资料按钮
- **右侧 Tab 切换**：我的收藏（网格卡片，支持取消收藏）/ 浏览历史（时间线，支持清空）/ 账号设置（修改密码、通知开关、隐私设置）
- **空状态**：无收藏时显示空插画 + 引导文字

#### 3.2.14 关于我们（About）

- **左侧边栏 200px**：快速导航（网站简介 / 开发团队 / 技术栈 / 联系我们），点击平滑滚动
- **右侧主内容区**：项目背景与愿景、数据统计、开发者信息、技术栈图标展示、联系方式、反馈表单

#### 3.2.15 404 页（NotFound）

- Flex 居中布局，居中显示 404 大号文字 + "页面未找到" 提示 + 返回首页按钮

## 4. 核心非功能需求（NFR）

### 4.1 界面与视觉规范

| 规范项 | 具体要求 |
| :--- | :--- |
| 配色风格 | 清新明亮，主色调青绿/靛蓝，辅助色暖橙，背景浅灰白 |
| 字体规范 | 中文：PingFang SC / Microsoft YaHei；英文：Inter / 系统默认 |
| 图片规范 | 高质量，景点图 ≥ 1920×1080，美食图 ≥ 1200×800，统一圆角+阴影 |
| 间距规范 | 遵循 8px 网格，模块间距 24px/32px，卡片内边距 16px/24px |
| CSS 规范 | `<style scoped>` + BEM 命名（`.block__element--modifier`），组件根类名与组件 kebab-case 名一致 |

### 4.2 动效与交互规范

| 动效 | 实现方式 | 参数 |
| :--- | :--- | :--- |
| 轮播图 | Element Plus Carousel | 自动播放 5000ms，动画 500ms |
| 返回顶部 | Element Plus Backtop | 滚动 > 200px 显示，平滑滚动 300ms |
| 表单验证 | Element Plus Form Rules | blur 实时校验，错误 shake 300ms |
| 卡片悬停 | CSS Transition | translateY(-4px) + box-shadow，300ms ease |
| 图片悬停放大 | CSS Transform | scale(1.05)，400ms ease，overflow:hidden |
| 页面过渡 | Vue Router | 淡入淡出 opacity 0→1，200ms |
| 加载骨架屏 | Element Plus Skeleton | wave 动画 |
| 灯箱预览 | CSS + JS | 全屏遮罩，缩放过渡 |

### 4.3 响应式断点

| 设备 | 断点 | 布局调整 |
| :--- | :--- | :--- |
| 桌面端 | ≥ 1200px | 完整布局，侧边栏展开，网格最大列数 |
| 平板端 | 768px – 1199px | 侧边栏收起，网格减少列数 |
| 移动端 | < 768px | 单列布局，汉堡菜单 |

### 4.4 性能要求

- 首屏加载 ≤ 3s（3G 网络）
- 图片懒加载（非首屏图片滚动到可视区域再加载）
- 路由懒加载（动态 import，按页面拆分 chunk）
- 打包后 JS ≤ 500KB（gzipped）

### 4.5 数据存储

- 用户状态（登录信息、收藏列表、浏览历史）通过 Pinia + localStorage 持久化
- 收藏和历史记录按用户 ID 隔离存储，不同用户数据互不干扰
- 演示账号：`test / abc123`，首次启动自动初始化
